   ПРИЧИНА: Batch size был жестко задан на 30 отправителей
   РЕШЕНИЕ: Добавлены конфигурируемые параметры отправки
   КОМАНДА: python3 NewLinks.py → пункт 5 → выбрать 350

✅ ПРОБЛЕМА 3: "Ошибки timeout - нужны рабочие прокси"
   ПРИЧИНА: Без прокси Telegram блокирует мас-отправки
   РЕШЕНИЕ: Добавлена поддержка кастомных прокси с интерфейсом ввода
   КОМАНДА: python3 NewLinks.py → пункт 4 → добавить свои прокси

⚠️ ВАЖНОЕ:
═════════════════════════════════════════════════════════════════════════════

1. ПРОКСИ ОБЯЗАТЕЛЬНЫ ДЛЯ БОЛЬШИХ ОБЪЕМОВ
   Без прокси Telegram заблокирует после ~50-100 писем
   С прокси можно отправить 1000+ писем за раз

2. ГДЕ НАЙТИ РАБОЧИЕ ПРОКСИ:
   • Free proxy sites: но могут быть медленными
   • Платные сервисы: более надежно (рекомендуется)
   • Свои серверы: если есть доступ

3. КАЧЕСТВО ОТПРАВИТЕЛЕЙ:
   user0001@meta.ua и т.д. - это фейковые email'ы
   Они работают только если SMTP не требует верификацию пароля
   Если требует пароль - используй реальные аккаунты

4. ТАЙМАУТЫ И ЗАДЕРЖКИ
   Если видишь "timed out":
   • Первое: добавь прокси
   • Второе: увеличь timeout в NewLinks.py (строка ~191)
   • Третье: уменьши количество отправок в batch
